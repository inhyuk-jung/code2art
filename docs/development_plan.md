# Code2Art (Is My Code Art?) 개발 계획서

본 문서는 `code_to_art_idea.md` 기획안을 바탕으로 프론트엔드 중심의 PoC(Proof of Concept) 서비스를 완성하기 위한 단계별 개발 계획서입니다.

## 1. 프로젝트 핵심 요약
*   **목표**: 사용자 코드를 브라우저에서 분석하여 p5.js를 활용한 제너러티브 모던 아트로 렌더링.
*   **보안 원칙**: 100% 클라이언트 (브라우저) 기반 동작 (서버로 코드 전송 안 함).
*   **초기 기술 스택**: Next.js, TypeScript, Tailwind CSS, Framer Motion, Babel/Standalone, p5.js.

---

## 2. 마일스톤 및 단계별 개발 계획

### 🚀 Phase 1: 프로젝트 기반 설정 및 UI/UX 스캐폴딩
**목표**: 기본 프로젝트 환경을 구축하고, 메인 랜딩 페이지와 코드 입력 UI를 퍼블리싱합니다.

*   [ ] **환경 셋업**: Next.js (App Router), TypeScript, Tailwind CSS 초기 구성.
*   [ ] **디자인 시스템**: 갤러리/캔버스와 같은 부드러운 파스텔 톤 및 Neumorphism(Soft Pop) 스타일의 UI 컴포넌트(버튼, 카드, 모달 등) 구축.
*   [ ] **랜딩 페이지**: "Is My Code Art?" 컨셉을 강조하고 100% Local processing(보안)을 알리는 안내 섹션 제작.
*   [ ] **입력 UI**: 코드 입력을 위한 컴포넌트(textarea 확장 또는 Monaco Editor 경량 버전) 구현.
*   [ ] **마스코트 컴포넌트**: 귀여운 로봇 '코디(Cody)'의 상태별(대기, 분석 중, 결과 반응) 애니메이션 플레이스홀더 구성 (Framer Motion 또는 Lottie 웹 연동).

### 🧠 Phase 2: 코어 분석 엔진 구현 (AST & 메트릭)
**목표**: 브라우저 환경에서 코드를 구문 분석하고 수치(Seed 파라미터)를 추출합니다.

*   [ ] **AST 파서 연동**: `@babel/standalone` 또는 `acorn`을 활용하여 사용자 코드를 브라우저 위에서 파싱하는 엔진 적용 (초기 지원: JS/TS).
*   [ ] **코드 품질 지표 추출**: 파싱된 트리(AST)를 순회하며 다음 객관적 지표를 추출하는 로직 구현.
    *   **제어복잡도 (Cyclomatic Complexity)**: 조건문 및 반복문의 중첩도.
    *   **응집도/여백**: 함수의 길이, 단위 모듈의 크기 측정.
    *   **일관성/스멜**: 네이밍 컨벤션과 중복 코드, 매직 넘버 여부 감지.
    *   **의존성/결합도**: 전역 변수 참조, 파라미터 수 및 함수 간 참조 여부 감지.
*   [ ] **수치 정규화 (Normalization)**: 추출된 로우(raw) 데이터를 제너러티브 아트 렌더러가 사용하기 쉬운 0.0 ~ 1.0 사이의 배열(DNA/Seed) 값으로 변환.

### 🎨 Phase 3: 제너러티브 아트 렌더링 엔진 (p5.js) 통합
**목표**: 정규화된 Seed 데이터를 바탕으로 코드를 시각 예술로 변환합니다.

*   [ ] **p5.js 통합**: `react-p5`를 통해 아트 캔버스를 Next.js 컴포넌트 안으로 이식.
*   [ ] **화풍 1: 기하학적 추상 (몬드리안/바우하우스 풍)**
    *   복잡도와 결합도가 낮을수록 안정적인 수평/수직 분할과 조화로운 색상 적용.
*   [ ] **화풍 2: 서정적 추상 (칸딘스키 풍)**
    *   파라미터에 따라 유기적인 곡선, 원형 크기 및 불투명도 조절, 수채화 텍스처 느낌 적용.
*   [ ] **화풍 3: 액션 페인팅 (잭슨 폴록 풍)**
    *   Perlin Noise와 무작위성(Randomness)을 결합하여 동적이고 거친 선형/파티클 궤적 렌더링.
*   [ ] **캔버스 인터랙션**: 코드가 렌더링되는 실시간 애니메이션 효과(물감이 번지거나 선이 그어지는 과정) 추가.

### 📢 Phase 4: 공유 기능 및 최종 PoC 다듬기
**목표**: 결과물을 쉽게 저장하고 타인과 공유할 수 있도록 기능을 완성하고 배포합니다.

*   [ ] **이미지 추출**: 생성된 p5.js 캔버스를 고화질 형태의 PNG 이미지 파일로 다운로드하는 기능.
*   [ ] **소셜 공유 기능**: SNS(X, 페이스북, 쓰레드 등)에 쉽게 업로드할 수 있게 텍스트 및 프레임 클립보드 복사 등 바이럴 허브 연동.
*   [ ] **에러 핸들링**: 파싱이 불가능한 수준의 문법 오류 코드 입력 시 위트 있는 로봇의 피드백 (e.g. "코드에 치명적인 문법 오류가 있어 붓이 멈췄어요!").
*   [ ] **배포**: Vercel을 활용하여 PoC 환경 최종 릴리즈 및 모바일 반응형 테스트 완료.

## 3. 향후 브레인스토밍 (Post-PoC 확장)
PoC 검증 완료 후 도입될 수 있는 고도화 피처입니다.
1.  **AI (Stable Diffusion) 렌더링 도입**: 브라우저에서 추출된 Seed 메타데이터만을 백엔드로 전송해 렌더링. 원본 로직 유출 위험을 0으로 둔 채 초고해상도의 리얼 아트워크 반환 (Phase 2).
2.  **GitHub 리포지토리 연동**: 다중 파일, 폴더 구조 기반의 초거대 우주적 형태의 의존성 다이어그램 아트 워크 생성 지원.
3.  **다중 언어 확장**: JS/TS 외에 Python, Go, Java 등 코드를 분석할 수 있는 WebAssembly 기반(Tree-sitter 등) 파서 추가.
