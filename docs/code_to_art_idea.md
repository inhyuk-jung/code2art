# 프로젝트 기획: Code2Art (서비스 타이틀: Is My Code Art?)

## 1. 프로젝트 개요
- **목표**: 사용자가 작성한 단일 소스 파일(코드)을 분석하여, 코드의 논리적 아름다움을 **미술관에 걸릴 법한 모던 아트풍의 한 장의 그림 작품**으로 렌더링하는 서비스.
- **핵심 가치**: 정보의 단순한 차트나 그래프화(Data Visualization)가 아닙니다. 코드를 물감과 붓터치로 치환하여 개발자들에게 **"자신의 지적 결과물이 아름다운 조형적 예술로 승화되는 경험"**을 제공합니다. 이는 액자로 만들어 걸어두고 싶거나 SNS에 자랑할 수 있는 감성적 자극을 극대화합니다.

## 2. '코드의 아름다움' 평가 기준 및 시각화 매핑 (시각화 로직)
단순한 코드 포맷팅을 넘어, 소프트웨어 공학적인 관점에서 코드의 품질을 평가하고 이를 미술적 요소로 치환합니다.

### A. 제어 흐름 복잡도 (Cyclomatic Complexity)
- **개념**: `if`, `for`, `while`, `switch` 등의 분기문과 반복문이 얼마나 깊게 중첩되어 있는지 평가합니다.
- **시각화**: 선의 밀도, 레이어의 중첩, 형태의 파편화.
  - **Good (낮은 복잡도)**: 간결하고 정돈된 형태, 충분한 여백 (바우하우스 스타일의 미니멀리즘).
  - **Bad (높은 복잡도)**: 촘촘하게 뒤얽힌 선들, 화면을 가득 채우는 터뷸런스(Turbulence) 에너지.

### B. 모듈화 및 구조적 여백 (응집도)
- **개념**: 함수가 단일 책임 원칙을 잘 지켜 적절한 크기로 쪼개져 있는지 평가합니다.
- **시각화**: 구성의 구심점(Focal Point)과 여백의 배치.
  - **Good**: 명확한 시각적 중심이 존재하며 요소들이 조화롭게 응집됨.
  - **Bad**: 중심 없이 흩뿌려진 파편들, 혹은 숨 막히게 꽉 찬 God Object의 덩어리감.

### C. 일속성과 코드 스멜 (Consistency & Code Smells)
- **개념**: 네이밍 컨벤션의 일관성, 매직 넘버 사용 여부, 불필요한 코드 중복 존재 여부를 파악합니다.
- **시각화**: 색상 팔레트의 조화.
  - **Good**: 절제되고 조화로운 컬러 팔레트 사용 (모네/바우하우스 스타일).
  - **Bad**: 예측 불가능하고 무작위적인 색상 조합, 시각적 불협화음.

### D. 의존성 및 결합도 (Coupling)
- **개념**: 모듈(함수, 클래스) 간의 참조나 전역 변수 의존이 얼마나 복잡하게 얽혀 있는지 파악합니다.
- **시각화**: 도형들 간의 물리적 결속과 중첩 정도.
  - **Good**: 독립적인 오브제들이 깔끔하게 분리되어 부유함.
  - **Bad**: 요소들이 빽빽하게 맞물려 있거나 거미줄처럼 복잡하게 얽힌 형태.

## 3. 화풍 라인업 (Art Styles)
사용자는 코드를 예술로 변환할 때, 다음의 거장들로부터 영감을 받은 스타일을 선택할 수 있습니다.

- **[Auto] AI 자동 선택**: 코드의 메트릭(복잡도, 성향, 패러다임)을 분석하여 가장 잘 어울리는 화풍을 AI가 추천합니다.
- **바실리 칸딘스키(Abstract)**: 점, 선, 면의 기하학적 구성과 순수 추상.원색과 기하학적 도형의 조화.
- **반 고흐 (Van Gogh)**: 역동적인 소용돌이 붓터치와 강렬한 에너지.
- **모네 (Monet)**: 빛과 색채가 부드럽게 번지는 서정적이고 몽환적인 분위기.
- **달리 (Dalí)**: 시공간을 초월한 초현실주의적 구성과 정교한 묘사.
- **바스키아 (Basquiat)**: 거리의 에너지가 담긴 거칠고 즉흥적인 신표현주의 스타일.

## 4. 로직 구현: 하이브리드 아트 엔진
1. **코드 정밀 분석**: 브라우저 내에서 직접 AST를 분석하여 8가지 공학적 메트릭(Complexity, Cohesion, Consistency, Coupling, Depth, Readability, Sentiment, Paradigm)을 추출합니다.
2. **AI 프롬프트 엔지니어링**: 추출된 수치를 미술 비평 전문 용어로 치환하여 초고화질 이미지 생성 프롬프트를 빌드합니다.
3. **생성형 AI (DALL-E 3 / Gemini 2)**: 
   - **OpenAI DALL-E 3**: 프록시 서버를 통해 CORS 문제를 해결하며 최상의 예술적 퀄리티를 구현합니다.
   - **Google Gemini 3.1 Flash**: 최신 멀티모달 모델을 통해 빠르고 효율적인 작품 생성을 지원합니다.

## 5. UI/UX: 귀여운 아트 로봇 'ArtBot'
사용자가 코드를 입력하고 결과가 나오기까지의 과정을 지루하지 않게 **ArtBot**이 함께합니다.

- **Phase 0: 화풍 확정 (Thinking)**: 로봇이 코드를 돋보기로 살피며 고민하는 애니메이션.
- **Phase 1: 그림 그리는 중 (Painting)**: 로봇이 직접 팔레트와 붓을 들고 캔버스에 그림을 그리는 동적인 모션.
- **Phase 2: 도슨트 작성 중 (Writing)**: 로봇이 완성된 작품을 보며 감동적인 비평문을 작성하는 모습.

## 6. 보안 및 프라이버시
- **100% Local Code Parsing**: 사용자의 원본 코드는 절대 서버로 전송되지 않습니다. 브라우저 메모리 안에서만 분석이 이루어집니다.
- **Metric-only Transmission**: AI 이미지 생성을 위해서는 분석된 '수치값'과 '화풍 정보'만 전역 API(Proxy)를 통해 안전하게 전달됩니다.

## 7. 기술 스택
- **Base**: Next.js (App Router), TypeScript, Tailwind CSS
- **Animation**: Framer Motion
- **Parsing**: Babel Standalone (Client-side)
- **AI SDK**: OpenAI SDK, Google Generative AI SDK
- **Backend API**: Next.js API Routes (OpenAI Proxy for CORS)
